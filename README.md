# DailyLimit - GLM Coding Plan 终极抢购工具

自动抢购智谱AI GLM Coding Plan 的 CLI 工具。每天上午10:00补货，本工具采用**浏览器+API双管齐下**策略，9:45开始高频监控，一旦发现库存立即抢购并**自动创建订单**，支持**二维码扫码支付**和**新标签页支付**。

## 🔥 核心特性

- **🚀 双管齐下**: 浏览器刷新 + API轮询同时运行，谁先成功谁优先
- **⚡ 超高速**: API每秒轮询5次(200ms间隔)，比人工快100倍
- **🛡️ 防封策略**: 浏览器随机刷新，UA轮换，模拟人类行为
- **⏰ 智能时间**: 9:45自动登录，10:05自动停止
- **🎯 多套餐**: Lite/Pro/Max三种套餐，支持月/季/年订阅
- **💰 自动支付**: API检测到库存后**自动创建订单**，生成支付链接
- **📱 二维码**: 控制台显示支付二维码，手机扫码即可支付
- **🖥️ 新标签页**: 浏览器自动打开新标签页，不跳转当前页面
- **👀 可视化**: 浏览器窗口可见，实时监控抢购过程

## 环境要求

- Node.js >= 18.0.0
- npm >= 8.0.0
- Windows / macOS / Linux

## 安装步骤

### 1. 克隆仓库

```bash
git clone https://github.com/wsbjwjt/DailyLimit.git
cd DailyLimit
```

### 2. 安装依赖

```bash
npm install
```

### 3. 安装 Playwright 浏览器

```bash
npx playwright install chromium
```

### 4. 配置账号

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
BIGMODEL_USERNAME=你的用户名/邮箱/手机号
BIGMODEL_PASSWORD=你的密码

# 默认套餐 (lite | pro | max)
DEFAULT_PLAN=pro

# 默认周期 (monthly | quarterly | yearly)
DEFAULT_CYCLE=quarterly
```

## 使用方法

```bash

```

或运行混合模式（浏览器+API，自动打开新标签页）：

```bash

```

然后按提示选择套餐和周期即可。

## 技术方案对比

| 特性 | 浏览器方案 | API方案 |
|------|-----------|---------|
| **检测方式** | 按钮状态监控 | API库存查询 |
| **频率** | 随机刷新 | 200ms固定 |
| **触发时机** | 按钮变为可点击 | soldOut变为false |
| **订单创建** | 手动点击 | ✅ 自动创建 |
| **支付方式** | 当前页跳转 | ✅ 新标签页+二维码 |
| **优势** | 最直观，防封 | ✅ 最快，自动支付 |

## 防封策略

- ✅ **随机间隔**: 4-8秒随机，不形成规律
- ✅ **UA轮换**: 4种浏览器标识随机使用
- ✅ **滚动模拟**: 30%概率轻微滚动页面
- ✅ **随机暂停**: 10%概率暂停2秒模拟思考

## 常见问题

### Q: 抢购成功后会自动付款吗？
A: 不会自动付款。工具负责**抢到库存并创建订单**，付款需要你手动完成。但工具会提供二维码和支付链接，非常方便。

### Q: 二维码支付怎么用？
A: 抢购成功后，控制台会显示二维码，用手机微信/支付宝扫码即可支付。同时浏览器也会自动打开新标签页显示支付页面。

### Q: 为什么用两种方式？
A: **双保险**。API最快（200ms），浏览器最稳，两者同时运行提高成功率。API方案还能自动创建订单。

### Q: 会被封号吗？
A: 风险很低。浏览器有防封策略（随机刷新），API调用频率也在合理范围（每秒5次）。

### Q: 可以同时在多台电脑运行吗？
A: 不建议。可能导致账号被限制，建议只在一台设备运行。

### Q: 如果没抢到怎么办？
A: 工具运行到10:05自动停止。可以明天再试，或尝试切换不同套餐。

### Q: 如何确认产品ID？
A: 运行 `npx ts-node src/api-test.ts` 可查看实时产品列表。

### Q: 订单创建失败怎么办？
A: 工具会尝试多个订单创建API端点，如果都失败会自动切换到浏览器方案继续抢购。
